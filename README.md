# root-iyouxi-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可以是工具、展示页、小游戏、信息页或其他任意形式的静态网页。仓库本身不针对任何特定网站或域名，仅作为 HTML 页面的集中存储与版本管理。

## 项目简介

`root-iyouxi-pages` 是一个静态 HTML 页面集合仓库。每个页面在仓库中独立存在，便于单独访问、调试或分享。仓库结构简单，适合存放不需要后端支持的纯前端页面。

## 目录结构

```
root-iyouxi-pages/
├── index.html          # 仓库首页（可选，用于列出或导航到各页面）
├── page1/              # 示例页面目录（可自行创建）
│   └── index.html
├── page2/
│   └── index.html
└── README.md           # 本文件
```

每个页面建议放在独立文件夹内，以 `index.html` 作为入口文件，便于通过 GitHub Pages 或类似服务直接访问。

## 页面归档说明

- 所有页面均为纯 HTML / CSS / JavaScript 实现，不依赖后端服务。
- 页面内容不涉及具体商业、营销或诱导性信息。
- 归档目的为个人或项目使用，便于统一管理与版本回溯。
- 页面可能包含外部资源（如图片、字体、库文件），请确保引用来源合规。

## 维护说明

- 本仓库由维护者不定期更新，添加或修改页面。
- 欢迎通过 Issue 或 Pull Request 提交建议或修正，但需确保内容合法合规。
- 仓库不保证所有页面长期有效或兼容所有浏览器。
- 如需使用仓库内页面，请自行评估适用性。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在浏览器中直接打开任意页面的 `index.html` 文件即可预览。
3. 若启用 GitHub Pages，可在仓库设置中开启，并通过 `https://<用户名>.github.io/root-iyouxi-pages/页面目录/` 访问对应页面。
